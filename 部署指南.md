# 部署指南

本指南说明如何将本 Skill 部署到你本地的各类 AI 编程助手中使用。本 Skill 基于 Anthropic Claude Code 的 Skill 标准格式（`SKILL.md` + `references/` + `examples/`）编写，原生适配 Claude Code，也可适配其它主流 Coding Agent。

## 仓库结构

```
.
├── SKILL.md          # Skill 核心定义（触发条件、工作流程、输入/输出规范、质量约束）
├── references/       # 参考资料库（运行时被 Skill 按需读取）
├── examples/         # 模拟样例（展示标准输入与输出）
├── 部署指南.md        # 本文件
├── README.md         # 项目说明
└── LICENSE           # 版权声明
```

> 仓库名 / 目录名即对应 `SKILL.md` frontmatter 里的 `name` 字段。下文用 `<skill-name>` 指代本 Skill 的名称，用 `<你的用户名>` 指代你的 GitHub 用户名。

---

## 一、Claude Code（原生支持，推荐）

本 Skill 即为 Claude Code 的标准 Skill，无需转换，直接放入 skills 目录即可。

### 方式 A：个人级安装（所有项目通用）

把整个仓库克隆到 Claude Code 的个人 skills 目录：

```bash
git clone https://github.com/<你的用户名>/<仓库名>.git ~/.claude/skills/<skill-name>
```

安装后目录应为：

```
~/.claude/skills/<skill-name>/
├── SKILL.md
├── references/
└── examples/
```

### 方式 B：项目级安装（仅当前项目生效）

在目标项目目录下执行：

```bash
git clone https://github.com/<你的用户名>/<仓库名>.git .claude/skills/<skill-name>
```

### 使用方式

安装后在 Claude Code 中直接用自然语言描述需求，或使用 `SKILL.md`「触发条件 / 触发词」中列出的关键词，Claude 会自动加载并应用本 Skill。可在 Claude Code 中执行 `/skills` 查看与确认已安装的 Skill。

> 首次使用时，Claude 会在需要时自动读取 `references/` 与 `examples/`，无需手动复制内容。

## 二、OpenAI Codex CLI

Codex 通过 `AGENTS.md` 接收项目指令，并支持自定义 prompt。推荐两种方式：

### 方式 A：作为 Codex 自定义 Prompt（推荐）

把 `SKILL.md` 放入 Codex 的 prompts 目录：

```bash
mkdir -p ~/.codex/prompts
cp SKILL.md ~/.codex/prompts/<skill-name>.md
```

随后在 Codex 中通过 `/<skill-name>` 调用本 Skill。

### 方式 B：写入项目的 AGENTS.md（常驻上下文）

在项目根目录创建或追加 `AGENTS.md`：

```bash
# 在项目根目录
cat SKILL.md >> AGENTS.md
```

Codex 启动后会自动读取项目根目录的 `AGENTS.md`。若 `references/` 中的内容对任务关键，可一并整理进 `AGENTS.md`，或将其复制到项目并在 `AGENTS.md` 中注明路径。

## 三、Cursor

Cursor 使用 `.cursor/rules/` 目录管理规则（`.mdc` 格式，含 frontmatter）。

1. 在项目根目录创建 `.cursor/rules/<skill-name>.mdc`：
2. 写入以下内容（把 `SKILL.md` 的 `description` 与正文粘贴进去）：

```markdown
---
description: <把 SKILL.md 的 description 字段内容粘贴到这里>
globs:
alwaysApply: false
---

<SKILL.md 正文内容>
```

- `alwaysApply: false`：按需触发，等同于 Skill 的触发条件；
- `alwaysApply: true`：始终加载（适用于强约束场景）。

旧版 Cursor 也支持项目根目录的单文件 `.cursorrules`，直接把 `SKILL.md` 内容粘贴进去即可。

## 四、Windsurf

Windsurf 使用 `.windsurfrules` 文件：

```bash
# 项目根目录
cp SKILL.md .windsurfrules
```

全局规则可放在 `~/.codeium/windsurf/rules/` 下。

---

## 五、Cline（VS Code 扩展）

Cline 使用 `.clinerules` 文件或 `.clinerules/` 目录：

```bash
# 方式 A：单文件
cp SKILL.md .clinerules

# 方式 B：目录式（推荐，可管理多个 skill）
mkdir -p .clinerules
cp SKILL.md .clinerules/<skill-name>.md
cp -r references .clinerules/references
```

---

## 六、Gemini CLI

Gemini CLI 读取 `GEMINI.md`：

```bash
# 项目级
cp SKILL.md GEMINI.md
# 或追加到已有文件
# cat SKILL.md >> GEMINI.md
```

全局配置可放在 `~/.gemini/GEMINI.md`。

---

## 七、其它 Agent（通用方法）

如果你的 Agent 不在上述列表中，但支持通过文本文件注入指令（几乎所有 Coding Agent 都支持），通用做法：

1. 找到该 Agent 读取「规则 / 指令」的文件位置（通常是项目根目录的一个 `.md` 文件，或某个 `rules/` 目录）。
2. 将本仓库的 `SKILL.md` 内容粘贴进去。
3. 把 `references/` 下的内容复制到同目录，并在指令中说明引用路径，便于 Agent 按需读取。

---

## 验证部署是否成功

安装后，在 Agent 对话中输入 `SKILL.md` 触发词中的任一关键词。若 Agent 按本 Skill 定义的工作流程与输出规范响应，即说明部署成功。

如遇问题，可在 GitHub 仓库提交 Issue。
