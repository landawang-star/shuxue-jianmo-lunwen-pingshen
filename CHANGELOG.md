# 更新日志 (CHANGELOG)

## v1.2.0 - 创造性区分度与论文类型化评分（解决系统性偏差）

### 背景

本版本针对第二位 reviewer 用国赛论文实测后提出的两个深层问题进行改进：

> 1. **创造性给保守分**：Skill 给创造性 18 分，依据是"综合运用多种方法+有据可依"，这恰恰把"突破"认成了"熟练组合"。18 这个分对一篇国二水平的扎实方法组合也成立，Skill 区分不出 18（国二组合）和 19-20（国一超额）。疫情风险乘数这种没有现成文献可"依"的创新，正好被"有据可依"这条标准卡住，上不去。
>
> 2. **格式权重放大失真**：把论文压到 67 的真凶是格式(11分)和文献(9分)。参考文献 6 篇没到内置的 8 篇线，但机器学习类国一论文文献少是常态。Skill 把"6 篇"判成扣分项，把"图表偏少"（机器学习论文特点，结果以表格呈现）也扣分，导致方法创新型论文被系统性低估。

### 实测验证（用真实 ML 国赛论文）

用 CUMCM 2020 C题优秀论文（脑电信号识别，随机森林/SVM/S3VM/贝叶斯稀疏回归，典型机器学习型）实测，确认：
- 该论文图5个、表4个（图表偏少，符合ML论文特征）
- 创新在跨方法融合（贝叶斯稀疏回归+spike and slab先验做通道选择），但无大量"创新"自述词
- 旧机制确实会把这类论文的创造性压在18、格式文献双扣分

**结论：reviewer 两个反馈均成立，v1.1.0 未触及这两个深层问题。**

### 本次改进内容

#### 改进 1：新增"国一 vs 国二评分区分线"（references 6.6）--解决反馈1

- 建立明确分数段：**19-20 国一超额 / 16-18 国二扎实组合 / 13-15 省奖 / <13 一般**
- 定义"突破标志"（🅰无文献先例 / 🅱针对赛题特殊结构自创 / 🅲跨领域融合合理性论证 / 🅳结果显著优越）：命中任一即可上探19-20
- **反向自检**：无突破标志的熟练组合不超过18，避免把国二误判为国一
- 弱化"有据可依"：明确"有据可依不应机械理解为必须有文献支撑"，开创性建模无文献先例恰是超额创新硬标志

#### 改进 2：新增"论文类型识别与类型化评分"（references 6.7）--解决反馈2

- 识别三类论文：A传统建模型 / B数据分析型 / C机器学习型
- **对机器学习型(C类)的特殊调整**：
  - "图表偏少"不扣分（领域特征，评估表格/指标完整性即可）
  - "文献偏少"宽容（ML论文常引用算法库而非大量理论文献）
  - 创新识别重点转移到特征工程/模型改进/半监督引入
  - 格式+文献合计扣分使总分被压低时，必须标注"扣分是否源于论文类型特征而非真实缺陷"，提示系统性低估可能

#### 改进 3：SKILL.md 工作流程与评分升级

- **第一步新增"论文类型识别"**（必做，影响后续所有评分），报告基本信息新增类型字段
- **维度二创造性**：评分指导改为按6.6区分线判定，明确19-20/16-18/13-15/<13四档及突破标志
- **维度三结果表述**：新增ML论文"图表偏少不扣分"的类型化调整
- 格式表"参考文献不少于8篇"改为弹性表述
- 质量约束新增第7条（区分度纪律）、第9条（类型化评分纪律）
- "不应该做什么"新增3条反例：不机械要求有文献、不把国二误判国一、不对ML论文因图表/文献偏少扣分

### 改进效果验证

用改进后机制重新评审 2020C 脑电信号ML论文：图表偏少不再扣分、文献偏少不扣分、跨方法融合创新命中突破标志🅲可上探19-20。改进前后完整对照见 `examples/实测验证_2020C脑电信号ML国赛.md`。

### 致谢

感谢第二位 reviewer 用真实国赛论文实测后的精准反馈。其指出的"区分度"和"机器学习型论文系统性低估"两个问题，比上一轮的"识别不到创新"更深一层--前者是"识别到了但分给不对"，后者是"用错了尺子"。本次改进从评分区分度和论文类型化两个维度根治了这两个问题。

## v1.1.0 - 创新性识别机制升级

### 背景

本版本针对一位 reviewer 提出的理论层面疑问进行改进。reviewer 指出：

> "国一论文与国二、国三论文最核心的区别，往往体现在'超额创新'上--比如自创算法、跨领域模型融合、对经典方法的突破性改进等。而这些'非常规'的创新，是否能用一套预设的规则库来有效识别和量化？"
>
> "对于真正新颖的方法，规则库可能'没见过'，从而给出保守评分；规则库可能会倾向于奖励'符合预期模板'的论文，而非真正有突破的论文。"

### 实测验证（reviewer 的担忧成立）

为验证该反馈是否准确，我们用真实的 CUMCM 国一论文（2020 B 题"穿越沙漠"，同济大学，全国一等奖）对改进前的 Skill 进行了实测，发现两个真实问题：

1. **漏判超额创新**：该论文的核心创新是自创的"多重搜索算法"（针对"第0天补给策略影响初始状态"这一赛题特殊结构设计）。但旧规则库的创造性评分指导只有"模型有明显创新或改进 -> 高分"的纯语义判断，缺乏识别机制。更关键的是，**作者几乎不用"创新/改进/提出"等词自我标榜**（全文仅1处"改进"且是句尾），创新全藏在方法组合里，靠关键词匹配根本捕捉不到。

2. **机械扣分惩罚真实国一**：旧规则库规定"参考文献不少于8篇"，而该真实国一论文**仅4篇参考文献**。按旧规则，格式维度会因此被扣分--但实际上这是方法创新型论文，文献数量本可少于综述型。

**结论：reviewer 的反馈完全准确，且问题比其设想的更具体。**

### 本次改进内容

#### 改进 1：新增"创新性识别清单与置信度机制"（references/国赛评审标准.md 第六章）

在国赛评审标准中新增核心章节，提供可操作的识别机制（而非空泛承诺）：

- **6.1 超额创新的五种类型**：自创算法 / 跨领域模型融合 / 经典方法突破性改进 / 建模视角创新 / 多方法协同框架。评审时先判断论文命中哪类。
- **6.2 创新信号识别清单（7项 S1-S7）**：针对性、动机说明、方法组合、改进点明确、验证支撑、自创命名、结果优越。评分前必填，命中越多越倾向高分。
- **6.3 创新性置信度评级**：高/中/低三级，写入评分表。低置信度时给出保守下限分而非直接压分。
- **6.4 人工复核触发条件（强制）**：当置信度低、出现自创命名算法、涉及跨领域融合时，**必须**在报告里追加人工复核建议，交由具备领域知识的评委裁定。
- **6.5 实例参考**：基于真实国一论文（穿越沙漠）整理创新信号特征，供识别时参照。

#### 改进 2：修正"参考文献不少于8篇"的刚性规则

将"不少于8篇"从硬性要求改为弹性规则：**方法创新型论文**（核心创新在自创算法/模型改进而非文献综述）允许少于8篇，此时不因数量扣分。避免对真实国一论文的误伤。

#### 改进 3：SKILL.md 评审流程与输出模板升级

- "维度二：模型建立的创造性"评分指导改为**强制读取 references 第六章**，按"五类型 -> 信号清单 -> 置信度 -> 人工复核"流程执行。
- 分项评分表新增**"置信度"列**。
- 报告模板"4.2 模型建立的创造性"节新增**创新信号识别清单、超额创新类型、置信度、人工复核位**。
- 质量约束新增第6、7条：创新性评审纪律、弹性规则适用。
- "不应该做什么"新增3条反例：不因未用"创新"等词而低估、不因方法名"没见过"就保守给分、不对低置信度评分直接定终分。

### 改进效果验证

用改进后机制重新评审 2020B 穿越沙漠国一论文：

| 评审项 | 改进前 | 改进后 |
|--------|--------|--------|
| 多重搜索算法识别 | ❌ 可能保守评分 | ✅ 命中 S1/S2/S6，按超额创新①识别，给高分 |
| 参考文献仅4篇 | ❌ 机械扣分 | ✅ 判定方法创新型，弹性适用，不因数量扣分 |
| 创新隐藏在方法组合 | ❌ 关键词匹配漏判 | ✅ 读完建模章节对照清单识别 |
| 置信度提示 | ❌ 无 | ✅ 标注置信度中，触发人工复核建议 |

### 致谢

感谢该 reviewer 的深度反馈。其提出的"超额创新难以规则化"的疑问，经实测验证后确实成立，促使本 Skill 从"纯规则驱动"升级为"规则 + 信号识别 + 置信度 + 人工复核"的混合评审机制，显著提升了对真实国一论文创新的识别能力。
