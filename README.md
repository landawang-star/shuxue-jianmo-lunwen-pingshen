# 数学建模国赛论文评审助手

> 按照中国大学生数学建模竞赛（CUMCM）国奖评审标准，对参赛论文进行全面审核，输出**分项评分表 + 格式问题清单 + 修改建议报告**的 Skill。让评委、指导教师、参赛团队都能用同一把"国奖标尺"快速、客观地衡量一篇论文。

## 一、这个 Skill 具体能帮你干啥？

一句话：**把一篇数学建模竞赛论文丢给它，它按国奖标准帮你"过一遍筛子"，告诉你这篇论文到底处于什么水平、哪里扣分、怎么改才能往上够一够国奖。**

具体到三类典型用户：

### 🧑‍⚖️ 给竞赛评委：批量化、标准化审核
- 一次评审要看的论文多、维度杂，靠人工逐篇核对格式和评分既慢又容易因疲劳漏判
- 本 Skill 把评审拆成"格式 6 项 + 内容 5 维"，逐项自动核对并打分，让评委把精力放在**有争议的关键判断**上，而不是重复的格式检查

### 👨‍🏫 给指导教师：赛前预审，少走弯路
- 学生提交论文前，先用它跑一遍，提前暴露"图表没编号""参考文献不足 8 篇""摘要超字""模型假设没依据"等低级失分点
- 拿到的不只是分数，还有**按优先级排好的修改清单**，可以直接转给学生照着改

### 👨‍🎓 给参赛团队：自查"能不能冲国奖"
- 比赛圈里最常见的问题就是"我们这篇到底什么水平，能不能拿国奖"
- 本 Skill 不替你做"一定/不能拿奖"的绝对判断（那是评委会做的），但会**告诉你距离国奖水平还差在哪、差多少**，让备赛有明确的发力方向

## 二、里面有什么功能？

### 📥 1. 多格式论文读取与结构解析
- **支持格式**：Word（.docx）、PDF（.pdf）、Markdown（.md）
- **结构完整度检查**：自动识别论文是否包含国奖论文应有的 8 个标准章节--摘要、问题重述、模型假设与符号说明、模型建立与求解、模型检验与分析、模型评价与推广、参考文献、附录，逐项打勾确认

### 📐 2. 格式规范性审核（6 个维度）
按国奖要求逐项检查，每个问题都记录**「维度 → 问题描述 → 位置（章节/页码/行号）→ 严重程度 → 修复建议」**：

| 检查维度 | 检查要点 |
|---------|---------|
| 标题与段落 | 标题编号统一、不跳级，段落间距与缩进规范 |
| 图表编号 | 图表按序编号、正文有引用、编号一致 |
| 公式规范 | 公式居中、编号右对齐、变量首次出现有定义 |
| 参考文献 | GB/T 7714 格式、文中标记与文末列表一一对应、不少于 8 篇 |
| 摘要规范 | 300-500 字、含问题/方法/结果/结论、关键词 3-5 个 |
| 页面规范 | 页眉页脚、页码连续、正文字号不小于小四 |

### 🎯 3. 内容质量审核（5 大评审维度，含权重打分）
对照国奖评审标准，逐项分析并打分（每项 0-20 分，合计换算为 100 分）：

| 评审维度 | 权重 | 看什么 |
|---------|------|--------|
| 模型假设的合理性 | 20% | 假设有没有依据、是否与模型方法一致、求解中有没有被忽略 |
| 模型建立的创造性 | 25% | 有没有针对性创新或改进、是否综合运用多种方法、选型有没有依据 |
| 结果表述的清晰性 | 25% | 图表是否规范、结论有没有定量数据支撑、结果是否直观易懂 |
| 论文格式的规范性 | 15% | 汇总格式审核结果给出规范度评分 |
| 参考文献与引用 | 15% | 文献覆盖面、引用标注是否准确、有无大量漏标 |

### 📋 4. 结构化评审报告输出
最终交付一份完整报告，包含：
- **基本信息**：论文标题、篇幅、章节完整度（X/8）
- **格式问题清单**：逐条问题 + 位置 + 严重程度（致命/一般/轻微）+ 修复建议
- **分项评分表**：5 个维度各自的得分与评分理由（不只给分，还说明为什么）
- **内容评审意见**：每个维度的优点与不足具体分析
- **综合评价**：整体印象、主要亮点、关键不足、与国奖水平的差距
- **修改提升建议**：按 P0（必须改）/ P1（建议改）/ P2（可选优化）三级优先级排序，指向论文中可操作的具体改动

### 🔧 5. 可自定义评审标准
- 默认使用内置国赛评审标准（见 `references/国赛评审标准.md`）
- 若用户提供了特定赛题或学校的评审细则，**优先使用自定义标准**，适配省赛、校赛或特定题目

### 🎚 6. 可指定审阅重点
- 用户可指定只看某个维度，如"只审核格式"或"重点看模型创新"，避免在不需要的地方浪费时间

## 三、是按照什么样的逻辑来的？

本 Skill 的运作遵循一条清晰的评审流水线，**模拟一位严谨评委的真实审稿过程**：

```
① 读论文 → ② 查格式 → ③ 评内容 → ④ 打分数 → ⑤ 出报告 → ⑥ 给建议
```

### 第一步：论文读取与结构解析
完整读取论文全文，先确认"骨架"是否齐全--摘要、问题重述、模型假设、模型建立、模型检验、模型评价、参考文献、附录这 8 个标准章节是否都在。**章节缺失本身就是扣分项**，先记录下来。

### 第二步：格式规范性审核
进入"逐条体检"模式：把格式拆成 6 个维度，对照国奖要求一项一项查。每发现一个问题，就按 `维度 → 问题描述 → 位置 → 严重程度 → 修复建议` 的固定格式记录，**问题可定位到具体章节、页码甚至行号**，而不是泛泛说"格式不规范"。

### 第三步：内容质量审核
对照国奖评审五大维度（假设合理性 / 建模创造性 / 结果清晰性 / 格式规范性 / 参考文献），**逐项分析论文的实际内容**：模型假设有没有依据、建模方法有没有创新、结果有没有数据支撑、文献覆盖够不够。这一步是评审的核心，决定论文的真实水平。

### 第四步：分项评分
基于上一步的分析，每个维度给出 0-20 分的评分，**每项分数都附带评分理由**（引用论文中的具体内容或评审标准作为依据），杜绝"只给分不解释"。五项加总换算为 0-100 分。

### 第五步：生成评审报告
把前面所有发现整合成一份结构化报告：基本信息、格式问题清单、分项评分表、内容评审意见、综合评价，层次分明，可读性强。

### 第六步：修改提升建议
最后，从所有问题中提炼出可操作的修改建议，按 **P0 / P1 / P2 三级优先级**排序，每条建议都指向论文中的具体位置和具体改动方向--不说"多查文献"这种空话，而是说"第 3.2 节公式 (5) 的变量 α 未定义，需在公式前补充说明"。

### 贯穿全程的两条红线（质量约束）
1. **不编造内容**：所有评分和意见必须基于论文实际内容，不凭空捏造数据、假设或结论
2. **不做最终判决**：不输出"这篇一定/不能拿国奖"的二元判断，只给达标程度分析和差距描述--因为最终获奖与否由评审委员会综合多因素决定，本 Skill 只做客观水平评估
